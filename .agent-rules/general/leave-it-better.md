# Leave It Better Than You Found It

## Mandatory: Track Discovered Issues

When working on any task, if you discover any of the following, you MUST add them as items in
your plan or task list (e.g., AGENTINFO.md, a plan file, or your active task tracker):

- Bugs (broken logic, runtime errors, incorrect behavior)
- Failing or missing tests
- Lint violations or type errors
- Broken imports or dead code
- Stale documentation that contradicts the code
- Rough edges in UX, error messages, or developer experience

## Priority and Scope

- Prioritize discovered issues relative to your current task — they do not need to be fixed first,
  but they MUST be tracked and fixed before you declare your work complete.
- If an issue is unrelated to your current work, note it in AGENTINFO.md under a "Discovered Issues"
  section so it is not lost.
- If it IS related to your current work, add it to your active plan and fix it as part of this session.

## The Standard

You ALWAYS leave the codebase better than you found it. If you found it broken, ugly, or with
rough edges — you fix those things. Work is not complete until discovered issues are resolved.
